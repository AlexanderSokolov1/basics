1           Задание

Note: В качестве базы данных используйте любую доступную вам базу. Это может быть общедоступная база NothWind или любая база из курса по проектированию баз данных.

Note: Выбранная для работы таблица должна содержать несоставной Primary Key, желательно (но не обязательно) числового типа.

1.1       Основное

Имея базу данных с любой таблицей, создайте следующие классы:

·        Класс для описания сущности таблицы

o   В качестве имени класса используйте название вашей таблицы.

o   У класса должны быть публичные авто-свойства, которые представляют колонки выбранной вами таблицы.

·        Класс MyTableService (где MyTable имя выбранной вами таблицы), содержащий методы для CRUD операций над таблицей:

o   Get(ключ) – возвращает запись из таблицы (объект класса MyTable) в базе данных по ключу.

o   Get(int top) – возвращает top записей (List<MyTable>) из таблицы в базе данных. Если top <= 0, то возвращает все записи.

o   Insert(MyTable) – вставляет сущность в таблицу в базе данных. Метод должен вернуть true, если получилось добавить запись и false – если нет (например, когда в базе уже есть запись с таким ключом).

o   Update(ключ, MyTable) – обновляет все колонки записи в базе данных с ключом «ключ» на значения полей переданного объекта.

o   Delete(ключ) – удаляет запись из таблицы в базе данных. Метод возвращает true если запись удалена и false, если такой записи не было.

В качестве технологии использовать присоединённый режим ADO.NET.

Все параметризованные запросы должны оформляться через параметры команды.

1.2       Дополнительное

Создать консольное приложение, демонстрирующее работу созданного класса MyTableService. Реализовать обработку всех возможных исключений, чтобы при ошибке приложение не аварийно завершалось, а отрисовывало в консоль сообщение об ошибке. Предоставить пользователю возможность ввода полей для формирования сущности записи вашей таблицы (при выполнении операции Insert и Update

1.4       задание со звёздочкой

Необязательное задание.

·        Реализовать возможность автогенерации ключа на стороне SQL сервера используя при формировании ключевой колонки ключевое слово IDENTITY.

·        Доработать MyTableService следующим образом:

o   Метод Insert не должен рассчитывать на то, что в объекте MyTable будет проинициализировано свойство с ключом.

o   При вставке новой записи метод Insert НЕ должен передавать ключ в качестве параметра запроса, заставляя тем самым SQL сервер сгенерировать этот ключ.

o   Метод Insert должен возвращать вставленную сущность с инициализированным свойством ключа. Не посылать отдельный запрос для получения сгенерированного ключа. Для его получения ознакомьтесь с вариантами в интернете (например тут).
